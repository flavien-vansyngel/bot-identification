Identifying bot session¶

Aim - From a set of session features, identify which are coming from humans and which are coming from bots.

Input - The two CSV tables containing the sesions features (including "request interarrival time", "request interarrival time" and "number of sessions") as computed and written by the notebook ComputeFeatures.ipynb.

Output - Separation of the clients, presumably between humans and bots.

Method - Compute an average of the features for each client then perform a clustering of the clients (k-means with 2 clusters). The goodness of clustering is evaluated with a silhouette plot. The 2 clusters are identified as human or bots given the average values of the features.

Tools - SparkSQL and SparkML from Spark 2.4.3, pandas 0.24.2, matplotlib 2.2.2

Average the features ¶

Compute the average of the features for each client.
In addition to the three features of the paper, the total number of sessions is added to the list of features.

In [5]:
avg_ss = sessions.where(col('session_interarrival_time').isNotNull()) \
                 .groupBy('client')                                   \
                 .agg({'number_of_requests':'mean',
                       'session_interarrival_time':'mean',
                       'session_number':'max'})                       \
                 .withColumnRenamed('avg(number_of_requests)', 'avg_number_of_requests')               \
                 .withColumnRenamed('avg(session_interarrival_time)', 'avg_session_interarrival_time') \
                 .withColumnRenamed('max(session_number)', 'total_number_of_sessions')

avg_rq = requests.where(col('request_interarrival_time').isNotNull()) \
                 .groupBy('client')                                   \
                 .mean('request_interarrival_time')                   \
                 .withColumnRenamed('avg(request_interarrival_time)', 'avg_request_interarrival_time')
    
avg_ft = avg_ss.join(avg_rq, on='client', how='inner')

Pre-process and format the features ¶

Take the log of the features because their distributions are heavy tailed (cf. result plots below).
Replace the client by an ID number.

Plot of projections of the clusters.
Two clusters are identified. The clustering mainly relies on the request interarrival time feature, with a separation around 15 to 20 sec.
Since bots usually send requests at the highest possible frequency (with tricks like random time interval to avoid being detected), we suppose that the cluster with lowest request interarrival times (yellow in the html version of the notebook) is the bot cluster.
The plots with the added feature total number of session exhibits interesting structures to investigate (like preferred constant times).
